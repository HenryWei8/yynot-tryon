# YYNOT? Virtual Try-On

A virtual try-on web app powered by FASHN VTON v1.5.

## Quick Start

```bash
# Generate catalog (after adding assets)
python3 gen_catalog.py

# Run pre-renders (CPU — slow; GPU recommended)
./prerender.py
# or for specific pairs:
./prerender.py --models model1 --garments shirt1

# Serve the app
python3 -m http.server 8080
# Open http://localhost:8080
```

## Directory Structure

```
yynot-tryon/
├── assets/
│   ├── models/      ← person photos (jpg/png/webp)
│   ├── garments/    ← flat-lay garment photos
│   └── renders/     ← inference outputs (auto-named model__garment.jpg)
├── fashn-vton-1.5/  ← inference engine (git clone)
│   └── .venv/       ← Python virtualenv
├── index.html       ← frontend (no build step)
├── catalog.json     ← generated by gen_catalog.py
├── prerender.py     ← batch inference runner
└── gen_catalog.py   ← catalog generator
```

## Adding Your Own Garments

1. Drop flat-lay garment images into `assets/garments/`
2. Drop model person images into `assets/models/`
3. Run `python3 gen_catalog.py`
4. Run `./prerender.py` to generate try-on renders
5. Refresh browser

## Naming Tips

- Include `short`, `pant`, `jean`, `trouser` in garment filename for auto-detection as bottoms
- Include `dress`, `jumpsuit` for one-pieces
- Everything else defaults to tops

## Keyboard Shortcuts

- `←` / `→` — previous / next garment
- `↑` / `↓` — jump 3 garments (grid row)
